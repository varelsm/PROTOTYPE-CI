# Plan: Redesign Modal Laporan Bulanan (finance.html)

## Problem
Modal "Laporan Bulanan" di finance.html punya 3 tab (Overview, Pesanan, Keuangan) yang tumpang tindih dengan:
1. Page `purchases.html` — sudah punya detail order, tracking, invoice, total pengeluaran
2. Page `finance.html` tab "Keuangan" — sudah punya tabel transaksi dan metode bayar
3. Antar tab itu sendiri — Overview line chart dan Keuangan bar chart keduanya menampilkan "pengeluaran per bulan" (data sama)

## Goal
Redesign isi 3 tab modal Laporan Bulanan agar **tidak overlap** dengan page lain dan **tidak overlap antar tab**, tetap semua dalam format rupiah (kecuali yang memang jumlah).

## Structure Baru

| Tab | Fokus | KPI Cards | Chart 1 | Chart 2 | Data Baru |
|---|---|---|---|---|---|
| **Overview** | Ringkasan bisnis — "Seberapa baik bisnis saya bulan ini" | Total Order, Total Pengeluaran, Rata-rata/Order, Top Supplier | Line chart: **Jumlah order per bulan** (12 bulan) | Doughnut: **Breakdown kategori belanja** (Beras, Minyak, Gula, Lainnya) | `monthlyOrdersData` — { label, orders } 12 bulan |
| **Supplier** | Performa supplier — "Dari mana saya belanja dan seberapa baik" | Total Supplier Aktif, % Pesanan Tepat Waktu, Rata-rata Rating, Pengeluaran Terbanyak | Bar chart: **Pengeluaran per supplier** (rupiah) | Doughnut: **Distribusi % order per supplier** | `supplierData` — { label, orders, spend, rating } |
| **Keuangan** | Detail keuangan — "Bagaimana kondisi uang saya" | Total Pengeluaran, Limit Kredit Sisa, Tagihan Jatuh Tempo, Pembayaran Terproses | Bar chart: **Total pengeluaran per bulan** (6 bulan terakhir, rupiah) | Line chart: **Utilisasi kredit tempo %** (6 bulan) + Doughnut: **Breakdown kategori belanja** | `monthlySpendData` — { label, value } 6 bulan; `creditData` — { label, used } |
