# Rencana: Laporan Bulanan (3 Halaman Dashboard)

## Masalah
Tombol "Laporan Bulanan" ada di 3 halaman (`overview.html`, `purchases.html`, `finance.html`) tapi **belum ada JavaScript handler** — tombol ini saat ini tidak melakukan apa-apa.

## Konteks Existing
- `purchases.html` baris 314: Card "Total Pengeluaran" → "Lihat Detail" sudah membuka modal `spendModal` dengan **Chart.js bar chart** interaktif, filter granularity (hari/minggu/bulan/tahun), export PNG, summary stats.
- `finance.html` baris 271: Sebelah tombol ini ada tombol "Unduh Laporan" yang berbeda.
- `overview.html` baris 235: Sebelah tombol ini ada "Restock Cepat".

## Solusi

Buat 3 **modal laporan bulanan** yang masing-masing menampilkan konteks berbeda sesuai halaman sumbernya. Pola modal sama seperti `spendModal` di `purchases.html` — slide-up (mobile) / centered overlay (desktop) dengan Chart.js, filter, export.

### 1. Overview → `Laporan Bulanan` (ringkasan bisnis)
**Halaman:** `overview.html`  
**Isi modal:**
- 4 summary cards: Total Belanja MTD, Pesanan Aktif, Poin GASPOLL, Limit Kredit Terpakai
- Chart: **Line chart** tren belanja 12 bulan terakhir
- Chart: **Doughnut chart** breakdown kategori belanja (Beras, Minyak, Gula, dll)
- Filter: Pilih bulan + tahun
- Tombol: Export PNG, Lihat Detail Pesanan → redirect ke `purchases.html`

**User flow:** Customer buka overview → klik "Laporan Bulanan" → dapat snapshot cepat performa bisnis bulan ini.

---

### 2. Purchases → `Laporan Bulanan` (ringkasan pembelian)
**Halaman:** `purchases.html`  
**Isi modal:**
- Summary cards: Total Pesanan MTD, Total Tagihan, Rata-rata per Order, Supplier Terbanyak Order
- Chart: **Bar chart** pesanan per minggu (sudah ada data `weeklyData`)
- Chart: **Pie chart** distribusi pesanan per supplier
- Filter: Pilih bulan + tahun
- Tombol: Export PNG, Lihat Semua Riwayat → redirect ke halaman order history

**User flow:** Customer di halaman purchases → klik "Laporan Bulanan" → dapat ringkasan pesanan bulan ini (bukan grafik pengeluaran — itu sudah di card "Total Pengeluaran → Lihat Detail").

**Pembedaan dengan modal existing:** Modal "Lihat Detail" di card Total Pengeluaran = fokus **pengeluaran/spending** (jumlah uang). Tombol "Laporan Bulanan" = fokus **order/pesanan** (jumlah order, supplier, kategori).

---

### 3. Finance → `Laporan Bulanan` (keuangan bulanan)
**Halaman:** `finance.html`  
**Isi modal:**
- 3 summary cards: Total Pengeluaran, Sisa Limit Kredit, Tagihan Jatuh Tempo
- Chart: **Bar chart** arus kas masuk vs keluar 6 bulan terakhir
- Chart: **Line chart** trend utilisasi kredit tempo
- Filter: Pilih bulan + tahun
- Tombol: Export PNG, Unduh Laporan PDF → redirect/trigger download

**User flow:** Customer di finance → klik "Laporan Bulanan" → dapat laporan keuangan komprehensif bulan ini.

**Pembedaan dengan tombol "Unduh Laporan":** Tombol "Unduh Laporan" di sebelah kanannya = trigger download PDF langsung. Tombol "Laporan Bulanan" = buka modal visual interaktif dulu.

---

## Implementasi Teknis

### File yang perlu dimodifikasi
1. `DASHBOARD/customer/overview.html` — tambahkan modal + JS
2. `DASHBOARD/customer/purchases.html` — tambahkan JS handler untuk tombol "Laporan Bulanan" (modal pembelian, BEDA dengan modal spendChart yang sudah ada)
3. `DASHBOARD/customer/finance.html` — tambahkan modal + JS

### Pola yang sama dengan existing `spendModal`
- Modal overlay: `fixed inset-0 z-[100]` dengan backdrop blur
- Container: slide-up di mobile, centered overlay di desktop (`md:top-10 md:left-10 md:right-auto md:max-w-[900px]`)
- Header dengan judul + close button
- Filter bar dengan dropdown (bulan, tahun)
- Chart container dengan canvas Chart.js
- Summary footer
- Export button

### Chart.js library
- `purchases.html` sudah load Chart.js (baris 8): `<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js"></script>`
- `overview.html` dan `finance.html` perlu ditambahkan script tag yang sama

### Data
- Gunakan mock data yang konsisten dengan data yang sudah ada di `purchases.html` (monthlyData, weeklyData, dll)
- Tambah mock data untuk finance (cash flow, credit trend) dan overview (kategori belanja)

### Struktur kode (reusable)
Buat 3 fungsi terbuka:
- `openOverviewReport()` → buka modal overview
- `openPurchaseReport()` → buka modal pembelian
- `openFinanceReport()` → buka modal keuangan

Masing-masing fungsi:
1. Tampilkan modal
2. Inisialisasi Chart.js sesuai konteks
3. Update summary stats

### Styling
- Gunakan class Tailwind yang sudah ada (color palette, fonts, spacing)
- Pastikan dark mode support (seperti existing `spendModal`)
